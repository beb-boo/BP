## 1. บทนำ

[ชื่อผู้ควบคุมข้อมูล/ชื่อบริษัท] ("เรา" หรือ "ผู้ควบคุมข้อมูล") เป็นผู้พัฒนาและให้บริการแพลตฟอร์ม BP Monitor ซึ่งเป็นเครื่องมือช่วยบันทึกค่าความดันโลหิตและเชื่อมต่อระหว่างผู้ใช้กับบุคลากรทางการแพทย์ ("บริการ") ผ่านทางเว็บแอปพลิเคชัน, Telegram Bot และ Telegram Mini App

เราตระหนักถึงความสำคัญของข้อมูลส่วนบุคคลและข้อมูลสุขภาพของท่าน นโยบายฉบับนี้จัดทำขึ้นเพื่อแจ้งให้ท่านทราบว่าเราเก็บรวบรวม ใช้ และเปิดเผยข้อมูลส่วนบุคคลของท่านอย่างไร ตามพระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 ("PDPA")

**ข้อมูลสุขภาพเป็น "ข้อมูลอ่อนไหว" (Sensitive Personal Data) ตามมาตรา 26 แห่ง PDPA** เราจะประมวลผลข้อมูลประเภทนี้เฉพาะเมื่อได้รับความยินยอมโดยชัดแจ้ง (Explicit Consent) จากท่านเท่านั้น

## 3. ข้อมูลที่เราเก็บรวบรวม

### 3.1 ข้อมูลส่วนบุคคลทั่วไป

- **ข้อมูลบัญชีผู้ใช้**: ชื่อ-นามสกุล, อีเมล, หมายเลขโทรศัพท์, รหัสผ่าน (เก็บในรูปแบบ hash เท่านั้น), ภาษาที่ต้องการ, เขตเวลา
- **ข้อมูลระบุตัวตน**: เลขบัตรประชาชน (สำหรับผู้ที่เลือกให้ไว้), เลขใบอนุญาตประกอบวิชาชีพเวชกรรม (สำหรับแพทย์)
- **ข้อมูลประชากร**: วันเกิด, เพศ
- **ข้อมูลบัญชี Telegram**: Telegram User ID (เมื่อท่านเชื่อมต่อบัญชี Telegram)
- **ข้อมูลการชำระเงิน**: หมายเลขอ้างอิงการโอนเงิน, จำนวนเงิน, ธนาคารผู้โอน, ชื่อผู้โอน, วันและเวลาที่ทำรายการ, ภาพสลิปการโอนเงิน

### 3.2 ข้อมูลสุขภาพ (ข้อมูลอ่อนไหว)

- **ค่าความดันโลหิต**: ค่า Systolic, Diastolic, ชีพจร (Pulse)
- **ข้อมูลร่างกาย**: กรุ๊ปเลือด, ส่วนสูง, น้ำหนัก
- **ข้อมูลการวัด**: วันที่และเวลาที่วัด, บันทึกเพิ่มเติม (notes)
- **ภาพถ่ายเครื่องวัดความดัน**: ภาพที่ท่านอัปโหลดเพื่อใช้ระบบ OCR อ่านค่าอัตโนมัติ

### 3.3 ข้อมูลทางเทคนิค

- **ข้อมูลการเข้าใช้งาน**: หมายเลข IP, ข้อมูลอุปกรณ์ (User Agent), Session Token
- **บันทึกการใช้งาน**: เวลาเข้าใช้, จำนวนครั้งที่ลงชื่อเข้าใช้ผิดพลาด, เวลาที่ใช้งานล่าสุด

### 3.4 ข้อมูลความสัมพันธ์แพทย์-ผู้ป่วย

- การเชื่อมโยงระหว่างบัญชีแพทย์และบัญชีผู้ป่วย
- คำขออนุญาตเข้าถึงข้อมูล (Access Requests) และสถานะการอนุมัติ
- ชื่อสถานพยาบาล

## 5. การเปิดเผยข้อมูลและผู้ประมวลผลข้อมูล

เราอาจเปิดเผยหรือโอนข้อมูลส่วนบุคคลของท่านให้แก่บุคคลภายนอกดังต่อไปนี้ ภายใต้วัตถุประสงค์ที่ระบุไว้ในนโยบายฉบับนี้:

### 5.1 ผู้ให้บริการภายนอก (Data Processors)

- **แพทยสภาแห่งประเทศไทย (TMC)**: เราส่งชื่อและเลขใบอนุญาตของแพทย์เพื่อตรวจสอบคุณสมบัติ ไม่มีการส่งข้อมูลผู้ป่วย
- **หน่วยงานของรัฐ**: เมื่อมีคำสั่งที่ชอบด้วยกฎหมายเท่านั้น

## 6. การโอนข้อมูลไปต่างประเทศ

เนื่องจากผู้ให้บริการภายนอกที่เราใช้บางรายมีที่ตั้งอยู่นอกประเทศไทย ข้อมูลส่วนบุคคลของท่าน **อาจถูกโอนไปยังต่างประเทศ** ได้แก่ สหรัฐอเมริกา สหรัฐอาหรับเอมิเรตส์ และสหราชอาณาจักร

เราดำเนินมาตรการเพื่อให้การโอนข้อมูลเป็นไปตามมาตรา 28 แห่ง PDPA ดังนี้:

- คัดเลือกผู้ให้บริการที่มีมาตรฐานการคุ้มครองข้อมูลส่วนบุคคลที่เพียงพอ
- ผู้ให้บริการทั้งหมดมีนโยบายความเป็นส่วนตัวและมาตรฐานความปลอดภัยระดับสากล (เช่น SOC 2, ISO 27001, หรือเทียบเท่า)
- ข้อมูลที่จัดเก็บในฐานข้อมูลถูก **เข้ารหัส (Encrypted)** ก่อนส่งออกจากเซิร์ฟเวอร์ของเรา ดังนั้นผู้ให้บริการฐานข้อมูลจะเห็นเฉพาะข้อมูลที่เข้ารหัสแล้วเท่านั้น ไม่สามารถอ่านข้อมูลดิบได้
- การสื่อสารทั้งหมดใช้โปรโตคอล HTTPS/TLS เพื่อเข้ารหัสข้อมูลระหว่างส่ง

## 7. มาตรการรักษาความปลอดภัย

เราใช้มาตรการทางเทคนิคและองค์กรเพื่อปกป้องข้อมูลส่วนบุคคลของท่าน ดังนี้:

### 7.1 การเข้ารหัสข้อมูล (Encryption)

- ข้อมูลส่วนบุคคลที่ระบุตัวตนได้ (ชื่อ, อีเมล, เบอร์โทร, เลขบัตรประชาชน, เลขใบอนุญาตแพทย์, วันเกิด, Telegram ID) ถูก **เข้ารหัสด้วย Fernet Symmetric Encryption (AES-128-CBC)** ก่อนจัดเก็บในฐานข้อมูล
- ฟิลด์ที่ต้องค้นหาจะถูก **แฮช (SHA-256 Hash)** แยกต่างหากเพื่อใช้ในการค้นหาโดยไม่ต้องถอดรหัสข้อมูลทั้งหมด
- รหัสผ่านถูกจัดเก็บด้วย **Bcrypt Hash** เท่านั้น เราไม่เก็บรหัสผ่านในรูปแบบข้อความธรรมดา

### 7.2 การควบคุมการเข้าถึง

- ระบบยืนยันตัวตนด้วย JWT Token พร้อมกำหนดอายุการใช้งาน
- การยืนยันตัวตน 2 ขั้นตอนผ่านรหัส OTP (ส่งทาง Email, SMS หรือ Telegram)
- ระบบล็อกบัญชีอัตโนมัติเมื่อลงชื่อเข้าใช้ผิดพลาดติดต่อกันเกินจำนวนที่กำหนด
- ระบบ Rate Limiting เพื่อป้องกันการโจมตีแบบ Brute Force
- การเข้าถึงข้อมูลผู้ป่วยโดยแพทย์ต้องได้รับการอนุมัติจากผู้ป่วยก่อน

### 7.3 ความปลอดภัยในการส่งข้อมูล

- การสื่อสารทั้งหมดระหว่างอุปกรณ์ของท่านกับเซิร์ฟเวอร์ใช้ HTTPS (TLS 1.2 ขึ้นไป)
- API ทั้งหมดต้องมี API Key ประกอบการเรียกใช้

### 7.4 ความปลอดภัยของภาพถ่าย OCR

- ภาพถ่ายเครื่องวัดความดันที่ท่านอัปโหลดจะถูกส่งไปยัง Google Gemini API เพื่อประมวลผล OCR เท่านั้น
- ภาพจะถูก **ลบออกจากเซิร์ฟเวอร์ทันทีหลังประมวลผลเสร็จ** โดยไม่มีการจัดเก็บถาวร
- เราไม่ใช้ภาพของท่านเพื่อฝึกโมเดล AI ของเรา

## 8. ระยะเวลาในการเก็บรักษาข้อมูล

| ประเภทข้อมูล | ระยะเวลา | เหตุผล |
|-------------|---------|--------|
| ข้อมูลบัญชีผู้ใช้ | ตลอดระยะเวลาที่บัญชียังใช้งาน + [90 วัน] หลังลบบัญชี | การให้บริการตามสัญญา |
| ข้อมูลสุขภาพ (ค่าความดัน) | ตลอดระยะเวลาที่บัญชียังใช้งาน หรือจนกว่าท่านจะลบ | ความยินยอม — ท่านลบได้ทุกเมื่อ |
| ภาพถ่าย OCR | ลบทันทีหลังประมวลผล | ไม่มีการจัดเก็บถาวร |
| รหัส OTP | หมดอายุอัตโนมัติภายใน 5 นาที | ความจำเป็นทางเทคนิค |
| ข้อมูล Session | หมดอายุตามกำหนด (สูงสุด 30 วัน) | ความปลอดภัย |
| ข้อมูลการชำระเงิน | 5 ปีนับจากวันทำรายการ | ภาษีและกฎหมายการเงิน |
| บันทึกความปลอดภัย (Logs) | [90 วัน] | ตรวจสอบเหตุการณ์ความปลอดภัย |

เมื่อครบกำหนดหรือเมื่อท่านใช้สิทธิลบข้อมูล เราจะลบหรือทำให้ข้อมูลไม่สามารถระบุตัวตนได้

## 9. สิทธิของท่านตาม PDPA

ท่านมีสิทธิตามกฎหมายดังต่อไปนี้:

**9.1 สิทธิในการเข้าถึง (Right of Access)**: ขอสำเนาข้อมูลส่วนบุคคลของท่านที่เราเก็บรวบรวมไว้

**9.2 สิทธิในการแก้ไข (Right to Rectification)**: ขอให้แก้ไขข้อมูลที่ไม่ถูกต้อง ไม่สมบูรณ์ หรือก่อให้เกิดความเข้าใจผิด

**9.3 สิทธิในการลบ (Right to Erasure)**: ขอให้ลบข้อมูลส่วนบุคคลของท่าน ทั้งนี้เราอาจไม่สามารถลบข้อมูลบางส่วนได้หากมีหน้าที่ตามกฎหมายที่ต้องเก็บรักษาไว้

**9.4 สิทธิในการระงับการประมวลผล (Right to Restriction)**: ขอให้ระงับการใช้ข้อมูลของท่านชั่วคราว

**9.5 สิทธิในการโอนย้ายข้อมูล (Right to Data Portability)**: ขอรับข้อมูลของท่านในรูปแบบที่อ่านได้ด้วยเครื่อง (เช่น CSV, JSON) เพื่อโอนไปยังผู้ให้บริการรายอื่น ท่านสามารถใช้ฟีเจอร์ "ส่งออกข้อมูล" (Export) ในแอปพลิเคชันได้ด้วยตนเอง

**9.6 สิทธิในการคัดค้าน (Right to Object)**: คัดค้านการประมวลผลข้อมูลที่อาศัยฐานประโยชน์โดยชอบด้วยกฎหมาย

**9.7 สิทธิในการถอนความยินยอม (Right to Withdraw Consent)**: ท่านสามารถถอนความยินยอมได้ตลอดเวลา การถอนความยินยอมจะไม่กระทบต่อความชอบด้วยกฎหมายของการประมวลผลที่เกิดขึ้นก่อนการถอน

**วิธีใช้สิทธิ**: ติดต่อเราทาง [privacy@yourdomain.com] เราจะดำเนินการภายใน 30 วัน นับแต่วันที่ได้รับคำขอพร้อมข้อมูลยืนยันตัวตนของท่าน

## 10. การประมวลผลข้อมูลในบริบทเฉพาะ

### 10.1 Telegram Bot และ Mini App

เมื่อท่านใช้บริการผ่าน Telegram:
- เราเก็บ Telegram User ID ของท่านเพื่อเชื่อมต่อกับบัญชีในระบบ
- ข้อความที่ท่านส่งถึง Bot (เช่น ค่าความดัน) จะถูกประมวลผลผ่านเซิร์ฟเวอร์ของ Telegram ก่อนส่งมาถึงระบบของเรา
- เราไม่สามารถควบคุมการเก็บข้อมูลโดย Telegram ได้ กรุณาตรวจสอบนโยบายความเป็นส่วนตัวของ Telegram ที่ [https://telegram.org/privacy](https://telegram.org/privacy)

### 10.2 การอ่านค่าจากภาพ (OCR)

เมื่อท่านอัปโหลดภาพเครื่องวัดความดัน:
- ภาพจะถูกส่งไปยัง Google Gemini API เพื่อสกัดค่าตัวเลข
- Google อาจประมวลผลภาพตามนโยบายการใช้ข้อมูลของ Google Cloud กรุณาตรวจสอบที่ [https://cloud.google.com/terms/data-processing-addendum](https://cloud.google.com/terms/data-processing-addendum)
- ภาพจะถูกลบจากเซิร์ฟเวอร์ของเราทันทีหลังประมวลผล

### 10.3 การตรวจสอบคุณสมบัติแพทย์

สำหรับผู้ลงทะเบียนเป็นแพทย์:
- เราส่งชื่อและเลขใบอนุญาตไปตรวจสอบกับแพทยสภาแห่งประเทศไทย (TMC) โดยอัตโนมัติ
- ข้อมูลผู้ป่วยจะไม่ถูกส่งไปในกระบวนการนี้

### 10.4 การชำระเงิน

- ภาพสลิปโอนเงินจะถูกส่งไปตรวจสอบกับ SlipOK API
- เราไม่เก็บข้อมูลบัตรเครดิตหรือข้อมูลบัญชีธนาคารของท่าน ระบบรับชำระเงินผ่านการโอนเงินเท่านั้น
