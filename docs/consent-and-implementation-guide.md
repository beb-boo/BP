# แบบให้ความยินยอมสำหรับข้อมูลสุขภาพ (Health Data Consent)

## สำหรับแสดงในแอปพลิเคชัน (ตอนลงทะเบียน/ตอนใช้ฟีเจอร์ OCR ครั้งแรก)

---

### ข้อความขอความยินยอม — ลงทะเบียน

> **ความยินยอมในการเก็บรวบรวมและใช้ข้อมูลสุขภาพ**
>
> ข้อมูลสุขภาพเป็น "ข้อมูลอ่อนไหว" ตามพระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 (PDPA) เราจึงขอความยินยอมจากท่านโดยชัดแจ้ง ก่อนเก็บรวบรวมและใช้ข้อมูลดังกล่าว
>
> **ข้อมูลที่เก็บรวบรวม:**
> ค่าความดันโลหิต (Systolic, Diastolic), ชีพจร, กรุ๊ปเลือด, ส่วนสูง, น้ำหนัก, วันและเวลาที่วัด, บันทึกเพิ่มเติม
>
> **วัตถุประสงค์:**
> 1. บันทึกและแสดงผลค่าความดันโลหิตของท่าน
> 2. วิเคราะห์แนวโน้มและสร้างกราฟสรุป
> 3. แชร์กับแพทย์ที่ท่านอนุมัติ (เท่านั้น)
> 4. ส่งออกข้อมูลตามคำขอของท่าน
>
> **ผู้ที่อาจเข้าถึงข้อมูล:**
> ข้อมูลถูกเข้ารหัสและจัดเก็บบนเซิร์ฟเวอร์ที่ตั้งอยู่ในสหรัฐอเมริกา (Neon, Vercel) แพทย์จะเข้าถึงได้เฉพาะเมื่อท่านอนุมัติ
>
> **สิทธิของท่าน:**
> ท่านสามารถถอนความยินยอมได้ตลอดเวลาผ่านการตั้งค่าบัญชีหรือติดต่อ [privacy@yourdomain.com] การถอนความยินยอมจะทำให้ท่านไม่สามารถใช้ฟีเจอร์บันทึกค่าความดันได้
>

## คำแนะนำในการนำไปใช้งาน (Implementation Guide)

### สิ่งที่ต้องทำเพื่อให้สอดคล้องกับ PDPA

1. **แสดงข้อความขอความยินยอมข้อมูลสุขภาพ (Explicit Consent) ตอนลงทะเบียน**
   - ต้องเป็น checkbox ที่ผู้ใช้ต้อง tick เอง (ห้าม pre-check)
   - ต้องแยกจากการยอมรับ Terms of Service (ไม่รวมเป็น checkbox เดียว)
   - ต้องมีลิงก์ไปยัง Privacy Policy ฉบับเต็ม
   - บันทึก timestamp และ IP ของการให้ความยินยอม

2. **แสดงข้อความขอความยินยอม OCR ก่อนใช้ฟีเจอร์ OCR ครั้งแรก**
   - ต้องให้ทางเลือก "ไม่ยอมรับ — บันทึกค่าด้วยตนเอง"
   - จดจำความยินยอมเพื่อไม่ต้องถามซ้ำทุกครั้ง

3. **บันทึกหลักฐานความยินยอม (Consent Log)**
   - เก็บบันทึก: ใครให้ความยินยอม, เมื่อไหร่, เรื่องอะไร, เวอร์ชันของข้อความที่แสดง
   - พิจารณาเพิ่มตาราง `consent_logs` ในฐานข้อมูล:
     ```sql
     CREATE TABLE consent_logs (
         id SERIAL PRIMARY KEY,
         user_id INTEGER REFERENCES users(id),
         consent_type VARCHAR(50) NOT NULL,    -- 'health_data', 'ocr_processing', 'doctor_access'
         consent_version VARCHAR(10) NOT NULL,  -- '1.0', '1.1' ฯลฯ
         granted BOOLEAN NOT NULL,
         granted_at TIMESTAMP NOT NULL DEFAULT NOW(),
         withdrawn_at TIMESTAMP,
         ip_address VARCHAR(45),
         user_agent TEXT
     );
     ```

4. **จัดทำหน้า Privacy Policy และ Terms of Service บนเว็บ**
   - เพิ่ม route `/privacy` และ `/terms` บน frontend
   - ลิงก์จากหน้าลงทะเบียนและหน้าตั้งค่า
   - ลิงก์จาก footer ของทุกหน้า

5. **ช่องทางถอนความยินยอมและใช้สิทธิ**
   - เพิ่มปุ่ม "ถอนความยินยอม" ในหน้าตั้งค่า
   - เพิ่มปุ่ม "ลบบัญชี" หรือ "ขอลบข้อมูล"
   - เพิ่มปุ่ม "ส่งออกข้อมูลของฉัน" (Data Portability — มีอยู่แล้วในระบบ)

### Placeholder ที่ต้องกรอก (ทั้ง 3 ไฟล์)

ค้นหาและแทนที่ `[...]` ในทุกไฟล์:

| Placeholder | คำอธิบาย | ตัวอย่าง |
|-------------|---------|---------|
| `[ชื่อผู้ควบคุมข้อมูล/ชื่อบริษัท]` | ชื่อจริงหรือชื่อนิติบุคคล | นาย พรเทพ xxx / บริษัท xxx จำกัด |
| `[ที่อยู่]` | ที่อยู่สำหรับติดต่อ | 123 ถ.xxx เขตxxx กรุงเทพฯ 10xxx |
| `[privacy@yourdomain.com]` | อีเมลสำหรับเรื่องข้อมูลส่วนบุคคล | privacy@bpmonitor.app |
| `[support@yourdomain.com]` | อีเมลสำหรับสนับสนุนทั่วไป | support@bpmonitor.app |
| `[https://yourdomain.com]` | URL เว็บไซต์หลัก | https://bpmonitor.app |
| `[วัน/เดือน/ปี]` | วันที่มีผลบังคับใช้ | 1 มกราคม 2569 |
| `[90 วัน]` | ระยะเวลาเก็บรักษาหลังลบบัญชี | ปรับตามความเหมาะสม |

### หมายเหตุสำคัญ

- เอกสารเหล่านี้เป็น **แนวทาง (Template)** ที่เขียนขึ้นตาม data flow จริงของระบบ BP Monitor แต่ **ไม่ใช่คำปรึกษาทางกฎหมาย** ควรให้ทนายความที่มีความเชี่ยวชาญด้าน PDPA ตรวจสอบก่อนเผยแพร่จริง
- ข้อมูลสุขภาพอยู่ภายใต้ PDPA มาตรา 26 ซึ่งมีบทลงโทษรุนแรง (ทั้งทางแพ่งและทางอาญา) หากไม่ปฏิบัติตาม
- หากมีแผนจะขึ้นทะเบียนเป็นอุปกรณ์ทางการแพทย์ในอนาคต จะต้องปรับข้อกำหนดเพิ่มเติมอย่างมาก
